C++

  1. Static 关键字的作用
    • static修饰函数内变量
      • static修饰的变量分配在静态存储区，在程序整个运行期间都不释放
      • static局部变量在初次运行时进行初始化工作，且只初始化一次
    • static修饰函数外变量/函数
      • 用来表示不能被其它文件访问的全局变量和函数
    • static修饰数据成员
      • 静态数据成员实际上是类域中的全局变量，这种数据成员的生存期大于类的对象
      • 静态数据成员每个类有一份，普通数据成员每个对象有一份
    • static修饰成员函数
      • 静态成员函数不可以调用类的非静态成员
      • 静态成员是可以独立访问的，也就是说，无须创建任何对象实例就可以访问
  1. const 与 #define的区别
    • 编译器处理方式不同
      • define宏是在预处理阶段展开
      • const常量是编译运行阶段使用
    • 类型和安全检查不同
      • define宏没有类型，不做任何类型检查，仅仅是展开
      • const常量有具体的类型，在编译阶段会执行类型检查
    • 存储方式不同
      • define宏仅仅是展开，有多少地方使用，就展开多少次，不会分配内存
      • const常量会在内存中分配(可以是堆中也可以是栈中)
  1. 指针和引用的区别
    • 引用在创建时必须初始化，引用到一个有效对象；
    • 引用一经声明，就不可以再和其它对象绑定在一起了
    • 引用的创建和销毁不会调用类的拷贝构造函数和析构函数
    • 指针在定义时不必初始化，可以在定义后的任何地方重新赋值
    • 指针可以是NULL，引用不行
    • 相对于指针，引用不用检查它所指对象是否为空，这增加了效率

  1. 函数指针

    • 函数是由载入内存的一些指令组成，而指向函数的指针存储了函数指令
      的起始地址
    • 声明：以下代码段的两种声明方式没有区别，从打印的地址相同可以看出;
      因为函数名在被使用的时候总是由编译器转化为函数指针。

  2. 虚函数(virtual function)

    • 虚函数用于实现多态
    • 在设计上具有封装和抽象的作用，比如抽象工厂模式。
    • 构造函数不可以是虚函数，析构函数可以是虚函数。
  1. 虚函数表
    • 编译器为每一个类维护一个虚函数表，每个对象的首地址保存着该虚函数表
      的指针，同一个类的不同对象实际上指向同一张虚函数表。
  1. 重载(overload), 覆盖(override) 和多态(polymorphism)
    • 重载(overload)是指在一个类中实现若干方法，这些方法有相同的函数名
      但是参数的形式或数目不同
    • 覆盖(override)是指子类在继承父类时，重写父类中的方法，其中函数名
      以及参数形式必须与父类中的方法一致
    • override不能是static同时必须是virtual的
    • 多态是为了避免父类中的大量重载写法，使得父类能够调用子类的同名方法。
  1. C++的多态
    • 多态分为静态多态和动态多态。静态多态是通过重载和模板技术实现，
      在编译的时候确定。动态多态通过虚函数和继承关系来实现，执行动
      态绑定，在运行的时候确定。
    • 静态多态：通过模板技术或者函数重载技术实现的多态，其在编译器
      确定行为。
    • 动态多态：通过虚函数技术实现在运行期动态绑定的技术。
    • 动态多态两要素：虚函数和一个基类的指针或引用指向派生类的对象
    • 基类指针在调用成员函数(虚函数)时，就会去查找该对象的虚函数表。
      虚函数表的地址在每个对象的首地址。查找该虚函数表中该函数的指
      针进行调用。
    • 每个对象中保存的只是一个虚函数表的指针，C++内部为每一个类维持
      一个虚函数表，该类的对象的都指向这同一个虚函数表。

Operating System

  1. 进程和线程
    • 调度：在引入线程的操作系统中，线程是调度的基本单位。而
      进程是资源拥有的基本单位。
    • 并发性：多进程和多线程都可以进行并发，不同场景下提高吞吐
    • 系统开销：进程的创建、撤销以及切换的开销远大于线程的开销
    • 通信：进程之间通信依靠IPC，线程间通信可以直接读写进程的数据
      段，如全局变量，然而需要同步互斥的辅助，来保证数据的一致性，
      所以线程的数据同步通常要比进程的复杂。
    • 综合比较：线程数据共享效率高，轻量，可应对并发操作，有效
      利用等待时间，但是多线程的编程要比多进程复杂，另外多进程的
      可靠性更高，因为进程间不会相互影响。一般情况，cpu-bound的
      任务使用多进程，而io-bound的任务使用多线程。
  1. 进程间通信IPC
    • 管道(pipe)：管道是一种半双工的通信方式，数据只能单向流动，
      而且只能在具有亲缘关系的进程间使用。进程的亲缘关系通常是
      指父子进程关系。
    • 命名管道 (FIFO) ： 有名管道也是半双工的通信方式，但是它允
      许无亲缘关系进程间的通信。
    • 信号量：信号量用于实现进程间的互斥与同步，而不是用于存储
      进程间通信数据，有XSI信号量和POSIX信号量，POSIX信号量更加完善。
    • 消息队列( message queue ) ： 消息队列是由消息的链表数组，存放
      在内核中并由消息队列标识符标识。消息队列克服了信号传递信息少、
      管道只能承载无格式字节流以及缓冲区大小受限等缺点。
    • 共享内存( shared memory ) ：共享内存就是映射一段能被其他进程
      所访问的内存，这段共享内存由一个进程创建，但多个进程都可以访问。
      共享内存是最快的 IPC 方式，它是针对其他进程间通信方式运行效率
      低而专门设计的。它往往与其他通信机制，如信号量，配合使用，来实
      现进程间的同步和通信。mmap建立进程空间到文件的映射，在建立的时
      ，同样采用缺页终端。mmap映射一个具体的文件可以实现任意进程间共
      享内存，映射一个匿名文件，可以实现父子进程间共享内存。候并不直
      接将文件拷贝到物理内存。
    • 套接字( socket ) ： 套解口也是一种进程间通信机制，与其他通信机
      制不同的是，它可用于不同及其间的进程通信。
