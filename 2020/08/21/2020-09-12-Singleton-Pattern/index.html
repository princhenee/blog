Singleton Pattern

Posted on 2020-08-21 | Edited on 2020-09-12

Why Singleton pattern?

There are cases where we need only a unique instance of the class. Examples are threadpool,
cache, dialogue, registry, log, device driver, system monitor, etc. If we create more than
one instances, the consequece may be abnormal program behavior, data inconsistency or resource
wastes. Therefore, Singleton Pattern is necessary to create a unique instance.

What’s the difference with Global Variable/Object?

If we use global object to solve the uniqueness problem, then we need to create an instance
every time before the program. Sometimes we may not need to create the instance, so global
object can be a waste of resource.

Implementation in Java

When to use?

  1. Eager initialization is easy to implement but it may cause resource and CPU time wastage. Use it only if cost of initializing a class is less in terms of resources or your program will always need the instance of class.
  2. By using Static block in Eager initialization we can provide exception handling and also can control over instance.
  3. Using synchronized we can create singleton class in multi-threading environment also but it can cause slow performance, so we can use Double check locking mechanism.
  4. Bill Pugh implementation is most widely used approach for singleton classes. Most developers prefer it because of its simplicity and advantages.
